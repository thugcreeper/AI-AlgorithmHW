問題:一直TLE (limit:1s,execution time:1.01~1.05)


原程式

int max_subarray(vector<int> vec,int low,int high)

這會導致每次遞迴呼叫max_subarray時都複製vector做為區域變數，而divide and conquer 會不斷呼叫，導致大幅增加記憶體和時間成本

解決方法:

  • 全域變數vector
  • call by reference(recommend)

不會TLE，時間約0.1秒完成

int max_subarray(vector<int> &vec,int low,int high)

前面有提到，如果直接寫 vector<int> vec 的話，程式會直接將每個內容物都複製一份出來 試想一下，如果每個 int 是 4 個 byte，假設陣列有 100000 個數字 那麼你每呼叫一次函式，記憶體就得處理 4 * 100000 bytes ≈ 0.4 MB 的資料 然後程式又呼叫了函式上百次…記憶體有太多東西要處理了 如果改成使用取址的方式，每次函式傳遞時只需要傳遞一個地址長度的資料 (32 bit 或 64 bit，看作業系統而定) ，明顯要處理的資料少多了。
